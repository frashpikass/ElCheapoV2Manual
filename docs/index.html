1. Introduction

Thank you for choosing ElCheapo!

ElCheapo V2.1 is a Game Boy flash cart with the following features:

  • 8MB of FLASH memory (4MB dedicated to the ROM and 4MB dedicated to the firmware).
    This flash memory can be overwritten a very high number of times (about 10000). It is slow to write and fast to read. Once a ROM image has been written to this memory, it is possible to run it countless times without need to rewrite it.

  • 128k of SRAM memory with no battery (fully available to the currently running software).
    This memory is overwritten with the latest savegame available every time you run the ROM from the flash memory.
    Since there is no battery, this memory is cleared every time the Game Boy is turned off.

  • microSD card slot
    The microSD card stores files to be loaded to the other two memories of the cart:
    • .gb files, for Game Boy ROM images to be loaded to the flash memory.
    • .gbc files, for Game Boy ROM images to be loaded to the flash memory.
    • .sav files, for common savefiles to be loaded to the SRAM.
    • .rtc files, to simulate internal clock status (used by ROMs like Pokémon Gold/Silver). !NEEDS FACT-CHECKING!
    • .state files, to store a full memory state dump (incompatible with some ROMs, including LSDj). !NEEDS FACT-CHECKING!

The cart is fully compatible with ROM images which use mbc1, mbc2, mbc3, mbc5, including LSDj.
Keep in mind that the cart does not support:

  • Internal battery to keep the saves across shutdowns
  • Internal real time clock (RTC)

2. Compatible Game Boy models
The ElCheapo v2.1 cart is not compatible with all Game Boy models.

It will not work correctly with some Game Boy Advance models and with some Game Boy Color motherboard revisions, especially 1, 3 and 4. In addition, Game Boy Color revision 1 has bugs in the sound generator section which make it unfit for chiptune production (LSDj included).

To easily discover the Game Boy Color motherboard revision one could look into the battery compartment and read the number under the bottom left negative contact.
If no 0X number is present or if you see only a couple of numbers like 1-1, your Game Boy Colour is a revision 01 and so it is not compatible with the ElCheapo v2.1.
Numbers like 02, 05 and 06 indicate compatible revisions.
You might even read the revision number in other parts of the motherboard, where it is written like CGB-CPU-0X, with 0X as the revision number.
You can find some Game Boy Color motherboard examples here.

Compatibility problems with Game Boy Advance and Game Boy Color revision 3 have been fixed since ElCheapo v2.2.
You can update your ElCheapo hardware for free (shipment cost excluded) by sending it to Ben Grimmett.

3. Initial setup
In order to setup your cart you will need:
  • A microSD card. It can either be SD or SDHC, but it must have a maximum size of 32 GB.
  • A computer with a microSD card reader or an analogous adapter.
The workflow is the following:
  1. Format the microSD card with a FAT32 file system. Choose the largest cluster size as possible.
  2. Create the SAVE directory at the root of the microSD card filesystem.
    That directory will store the savefiles generated by the ROMs executed from flash memory (keep in mind that to store a savegame here you will need to give a long push to the button on the back of the ElCheapo while the software is running).
  3. Copy the ROM files from the computer to any position inside the microSD card.
  4. Safely remove the microSD card from the computer, disabling it from the specific menu before phisically unplugging it from the computer.
It is now possible to delicately insert the microSD card inside the ElCheapo cartridge.
Make sure you insert it in the correct way (the main face of the microSD should facing you when you look at the cart label; also, the contacts should be in the general direction of the ElCheapo's contacts).

4. Use of the cart menu (Firmware 2.012)
Once the ElCheapo cartridge has been inserted into the Game Boy, after turning the console on you will interact with the firmware of the cartridge.

Splash screen

The splash screen is the first you will see after turning on the Game Boy.

  • Hitting the D-PAD, A or B will execute the ROM image which is currently available in the flash memory.
  • Hitting the SELECT button will open the SRAM menu. !NEEDS FACT-CHECKING!
  • Hitting the START button will open the main menu.

Main menu

The main menu is composed of 7 tabs, which you can select using the D-PAD and access with A.

  • Directory

    Allows the selection of the directory on which you want to work with the other main menu tabs.

    As with many common operating systems' file systems, the special directory called . indicates the current directory, while the one called .. indicates the parent directory of the current directory.

    • D-PAD is used to select a directory.
    • A opens the selected directory.
    • START returns the control to the main menu.
    • B returns to the splash screen. !NEEDS FACT-CHECKING!

  • GB ROM

    Allows to flash a Game Boy ROM image (a .gb file) from the ones available in the currently selected directory. !NEEDS FACT-CHECKING!

    • D-PAD is used to select a ROM image.
    • A overwrites the flash memory with the selected ROM image.
    • START returns the control to the main menu.
    • B returns to the splash screen. !NEEDS FACT-CHECKING!

  • GBC ROM

    Allows to flash a Game Boy Color ROM image (a .gbc file) from the ones available in the currently selected directory. !NEEDS FACT-CHECKING!

    • D-PAD is used to select a ROM image.
    • A overwrites the flash memory with the selected ROM image.
    • START returns the control to the main menu.
    • B returns to the splash screen. !NEEDS FACT-CHECKING!

  • Import SAV

    Allows the selection of a save file to use with the ROM image which is currently in the flash memory. Allows to flash a Game Boy ROM image (a .gb file) from the ones available in the currently selected directory. !NEEDS FACT-CHECKING!

    • D-PAD is used to select a save file.
    • A overwrites the save file in the SAVE folder of the microSD with the one selected.
    • START returns the control to the main menu.
    • B returns to the splash screen. !NEEDS FACT-CHECKING!

    !!! WARNING !!!

    In the SAVE folder on microSD you will find only a savefile for each ROM !NEEDS FACT-CHECKING!, which is the one that will be automatically loaded on boot (when you hit any key on the splash screen).
    That save file will be overwritten. It is advised to backup it on your PC before overwriting if it contains important data.

    !!! TIP !!!

    If you need only one version of LSDj for all your tracks in a liveset, you can store many different .sav files in the microSD and load them when needed without flashing the ROM image again.
    But first remember to back up your current save file in the SAVE folder if you have some work in progress!

  • State File

    !NEEDS FACT-CHECKING!

    Allows the selection of a state file to use with the ROM image which is currently in the flash memory. !NEEDS FACT-CHECKING!

    • D-PAD is used to select a state file.
    • A !NEEDS FACT-CHECKING!
    • START returns the control to the main menu.
    • B returns to the splash screen. !NEEDS FACT-CHECKING!

  • RTC File

    !NEEDS FACT-CHECKING!

    RTC files contain a simulated state for the internal real time clock of the cart (RTC).

    • D-PAD is used to select an RTC file.
    • A !NEEDS FACT-CHECKING!
    • START returns the control to the main menu.
    • B returns to the splash screen. !NEEDS FACT-CHECKING!

  • Firmware

    Allows the update of the ElCheapo's firmware with a .BEN file.
    You can download new firmware updates from this page.

    • D-PAD is used to select a .BEN firmware file.
    • A overwrites the current firmware in flash memory.
    • START returns the control to the main menu.
    • B returns to the splash screen. !NEEDS FACT-CHECKING!

    !!! WARNING !!!

    Updating the firmware is a delicate operation. To maximise your success rates it's advised to:
    • Check that the downloaded firmware version is correct. Don't install v1.x firmware on ElCheapo v2.x hardware!
      You can find your ElCheapo hardware version written on the on the front of the cart PCB; the current firmware version is displayed after moving from the splash screen to the main menu.
    • Make sure that the batteries of your Game Boy are fully charged before you update the firmware.
    • Don't turn off the Game Boy until the new firmware has been fully flashed.

    In case the update fails and your ElCheapo is bricked, you can fix it by reflashing it with an external writer like the Joey (available here).
    If you don't have a Joey, you can ask Ben to reflash your ElCheapo (you will have to provide for the shipping cost). !NEEDS FACT-CHECKING!

5. Running a ROM from the cart

To run the currently available ROM in flash memory using the save file in the SAVE directory of the microSD, hit any key (but START or SELECT) at the splash screen.

While the ROM is being executed, it is possible to update the corresponding save file in the SAVE directory of the microSD by pushing and holding the ElCheapo button on the back of the cartridge.

Some ROMs will support save states with the ElCheapo. A save state is a complete memory dump of the SRAM and the Game Boy RAM.
With compatible ROMs, you can create a save state by giving a fast push to the ElCheapo button on the back of the cartridge.

LSDj does not support save states and will crash if the ElCheapo button is pushed quickly.

6. Datasheet
These are the specifications available on the website for the ElCheapo v2.1
  • No RTC in this hardware version.
  • 8mbytes total, 4mbytes available for ROM
  • 128k total SRAM, 128k available to ROM
  • 3.3v to 5v level translators
  • No battery required.
  • SD not included. Tested with cards 1gb to 32gb SD+SDHC
  • Tested working on GBA, SP, GBC, DMG, Pocket. Save states tested on GBA, GBC, SP, DMG, pocket.
